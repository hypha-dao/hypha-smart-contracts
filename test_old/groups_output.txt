What this is

First I printed out the results of the edenia group_sizes - which creates the number of rounds, and then a number of groups per round dependent on that. 

Then I connected it to our code, where a number of ids comes in, and based on the number of troups caculated by edenia code, we create randomized groups for this round

The next round then contains as many members N as there were groups, meaning if there are 5 groups, then there's 5 winners, and the next round will be N = 5

In the first example, I set the group size to 100
The first round has 25 groups of size 4
The second round has 5 groups of size 5
The third round has 1 group of size 5

And so on - each round we calculate the number of winners and start over. 

We use the Edenia code to estimate number of rounds and group size at the beginning of each round.

It is not deterministic because a round may not have a winner.
